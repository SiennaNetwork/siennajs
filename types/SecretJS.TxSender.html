  • authz: { exec: SingleMsgTx<MsgExecParams>; grant: SingleMsgTx<MsgGrantParams>; revoke: SingleMsgTx<MsgRevokeParams> }
    • exec: SingleMsgTx<MsgExecParams>

      MsgExec attempts to execute the provided messages using authorizations granted to the grantee. Each message should have only one signer corresponding to the granter of the authorization.

    • grant: SingleMsgTx<MsgGrantParams>

      MsgGrant is a request type for Grant method. It declares authorization to the grantee on behalf of the granter with the provided expiration time.

    • revoke: SingleMsgTx<MsgRevokeParams>

      MsgRevoke revokes any authorization with the provided sdk.Msg type on the granter's account with that has been granted to the grantee.

  • broadcast: ((messages: Msg[], txOptions?: TxOptions) => Promise<Tx>)
      • (messages: Msg[], txOptions?: TxOptions): Promise<Tx>

      • Sign and broadcast a transaction to Secret Network.

        See

        • https://docs.tendermint.com/master/rpc/#/Tx/broadcast_tx_sync. If Async Broadcast transaction to mempool and do not wait for CheckTx response.
        • https://docs.tendermint.com/master/rpc/#/Tx/broadcast_tx_async.

        Parameters

        • messages: Msg[]

          A list of messages, executed sequentially. If all messages succeeds then the transaction succeed, and the resulting Tx object will have code = 0. If at lease one message fails, the entire transaction is reverted and Tx code field will not be 0.
